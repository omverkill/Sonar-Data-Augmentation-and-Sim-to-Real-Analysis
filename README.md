# Sim2Real Sonar Image Augmentation: A Comparative Study

A comparative evaluation of **diffusion-based** and **GAN-based** synthetic-to-real (Sim2Real) domain adaptation methods for side-scan sonar imagery, built to close the texture, noise, and acoustic-shadow gap between simulated sonar data (S3SIM) and real sonar datasets (SCTD, KLSG).

> **Repo contents:** this repository tracks the experiment notebooks only. Datasets and pre-computed metric outputs are **not** included — see [Datasets](#3-datasets) for sourcing and [Getting Started](#getting-started) for how to regenerate results locally.

## 1. Research Question

> Can diffusion-based or GAN-based augmentation bridge the Sim2Real domain gap for sonar object detection, and which family does it more effectively?

Real sonar datasets are small and costly to collect; simulator output (S3SIM) is cheap and unlimited but visually distinct from real acoustic imagery. This repo implements and benchmarks two competing families of methods for translating simulator output into realistic-looking sonar imagery, under a shared evaluation protocol.

## 3. Datasets

### 3.1 Real Sonar (Ground Truth)

| Dataset | Format | Images | Classes |
|---|---|---|---|
| SCTD 2.0 | Pascal VOC XML | 357 | ship (271), aircraft (57), human (35) |
| KLSG | Roboflow / YOLO | 521 train+val, 68 test | aircraft (105), fish (208), other (178), shipwreck (456) |
| **Combined Real** | — | **878–938**¹ | — |

¹ 878 images are used for SD1.5 LoRA training (SCTD + KLSG train/val); 938 is the merged `SIM2REAL/real/` set used for CycleGAN, Shadow-CycleGAN, SD2.1 LoRA, and StyleGAN2-ADA. KLSG carries a 4.3× class imbalance (aircraft is the minority class), which motivates CE-GAN (Section 4.2).

### 3.2 Synthetic Dataset

| Dataset | Source | Images | Classes |
|---|---|---|---|
| S3SIM | S3 Simulator | 572 | ship (313), plane (259) |

S3SIM is rendered at 224×224 (small) and 1920×972 (large) and is markedly brighter than real sonar (mean brightness ≈95 vs. ≈69). The domain gap is quantified via PCA centroid distance: **Synthetic→Real = 13.7**, vs. **Generated→Real = 5.4–5.5** after augmentation — roughly a 60% reduction, consistently observed across both GAN and diffusion outputs.

### 3.3 Obtaining the Data

None of the three datasets are bundled in this repository.

- **SCTD** is publicly released by its authors on GitHub: [`MingqiangNing/SCTD`](https://github.com/MingqiangNing/SCTD) (Pascal VOC XML annotations). Cite Ning et al. / Zhou et al. if you use it.
- **KLSG** (SeabedObjects-KLSG) is publicly available for academic use; it originates from Huo et al., *"Underwater Object Classification in Sidescan Sonar Images Using Deep Transfer Learning and Semisynthetic Training Data"* (IEEE Access, 2020). Several community mirrors exist — confirm you're pulling a version matching the class counts in §3.1 before reusing the captioning/parsing logic in `sd1.5_both.ipynb`.
- **S3SIM** is not confirmed publicly redistributable in this README — source it from wherever your S3 Simulator pipeline/export originated, and drop it into `SIM2REAL/fake/` once the directory structure is recreated locally (see [Getting Started](#getting-started)).

After downloading, recreate the local-only directories shown in [Repository Structure](#2-repository-structure): place SCTD under `SCTD/`, KLSG under `KLSG/`, merge both into `SIM2REAL/real/`, and place S3SIM under `SIM2REAL/fake/`.

## 4. Methods Compared

### 4.1 Diffusion-Based

**ControlNet — pre-fine-tuning baseline.** Off-the-shelf `lllyasviel/sd-controlnet-hed` / `sd-controlnet-canny`, run on S3SIM edge maps with the prompt *"sonar scan seabed underwater, grainy texture, acoustic shadow"*, with no domain-specific training. Establishes a baseline: a generic pretrained ControlNet has no notion of sonar acoustics and produces natural-image-like textures.

**ControlNet — post fine-tuning.** Same base (`runwayml/stable-diffusion-v1-5` + HED ControlNet) but with the ControlNet adapter fine-tuned on paired (S3SIM edge map → real sonar image) data; the SD1.5 UNet stays frozen. Structural layout (shadow shape, object outline) is preserved from the edge map while texture is adapted toward real acoustic noise patterns.

**SDEdit — SD1.5 LoRA.** SD1.5's UNet is LoRA fine-tuned (rank 8, alpha 8, 30 epochs, batch 4, cosine LR from 1e-4) on 878 captioned real images (SCTD + KLSG), then applied as img2img (SDEdit) directly on S3SIM images. Five inference variants per source image (`syn2real`, `syn2real_heavy`, `flipped`, `rot90`, `shifted`) at strengths 0.35–0.50 and guidance 6.5–7.5, producing 2,860 augmented images (1,565 ship + 1,295 plane).

**SDEdit — SD2.1 LoRA.** Higher-capacity variant: SD2.1 at native 768px, LoRA rank 64 across attention + feed-forward + conv layers, 60 epochs on 938 merged real images, with a 5-step strength/guidance schedule at inference (strength 0.45–0.65, guidance 6.0–7.0, 30 steps). Generates 1,565 ship images from the 313 ship sources. Checkpoints exist at epochs 25/30/35/40; convergence beyond epoch 40 is unconfirmed.

### 4.2 GAN-Based

**CycleGAN (100 / 200 epochs).** Standard unpaired image-to-image translation: 9-residual-block ResNet generators (11.38M params each) and 70×70 PatchGAN discriminators (2.76M each), trained both directions (S3SIM↔Real) with LSGAN + cycle (λ=15→10) + identity (λ=5) losses and a 50-image replay buffer. Extending training from 100→200 epochs (with cycle weight reduced to 10) improves FID by ~17 points.

**Shadow-CycleGAN.** Based on Kim et al. (2025){see [References](#12-references)}, this replaces the single-resolution ResNet generator with a multi-resolution UNet (4 decoder output heads at 32/64/128/256px) and a multi-resolution PatchGAN discriminator with feature fusion across branches, adding a **Shadow Area Loss** (K-Means, K=8, on the darkest cluster) so the model explicitly supervises the acoustic-shadow region that standard CycleGAN ignores. ~118.39M parameters across all sub-models; 200 epochs, λ_shadow=0.6.

**CE-GAN (Class-Equalizing GAN).** A conditional WGAN-GP used specifically to fix KLSG's 4.3× class imbalance by synthesizing 172 additional aircraft (Class 0) patches from only 89 real training patches. Generator (10.78M params) uses PixelNorm + spectral-norm upsampling; discriminator (11.22M params) adds self-attention and a dual adversarial/classification head. Training exhibited mode collapse between epochs 10–40, recovering by epoch 50; best checkpoint at epoch 25.

**StyleGAN2-ADA (transfer learning).** Fine-tunes from AFHQ-Wild pretrained weights onto 938 real sonar images (preprocessed to grayscale, histogram-stretched, and re-rendered as RGB at 256×256), with the first 10 discriminator layers frozen and ADA augmentation disabled (sufficient real data). Produces 508 novel images via W-space truncation (ψ=0.7), assembled into a combined 2,020-image augmented set (572 synthetic + 940 real + 508 generated).

## 5. Evaluation Framework

| Metric | Measures | Direction |
|---|---|---|
| FID | Distributional similarity, real vs. generated (InceptionV3, 2048-d) | lower = better |
| LPIPS | Perceptual realism, real vs. generated (AlexNet backbone) | lower = better |
| Diversity Score | Intra-set pairwise LPIPS | higher = less mode collapse |
| Cycle Consistency LPIPS | Reconstruction fidelity, A→B→A | lower = better |
| PCA / t-SNE | Feature-space domain-gap visualization | qualitative |
| Coverage | % of real images with an augmented neighbor (cosine dist ≤ 0.3) | higher = better |
| Density | Avg. augmented images near each real image | higher = better |
| Shadow IoU | Shadow-mask overlap, source vs. generated (Shadow-CycleGAN only) | higher = better |

## 6. Comparative Results

### 6.1 FID — Distributional Similarity (vs. Real B, 938 images)

| Method | Family | Epochs/Steps | FID ↓ |
|---|---|---|---|
| **CycleGAN (200 ep)** | GAN | 200 | **140.56** |
| CycleGAN (100 ep) | GAN | 100 | 157.37 |
| SDEdit SD1.5 LoRA — plane/syn2real_heavy | Diffusion | — | 215.28 |
| SDEdit SD1.5 LoRA — ship/flipped | Diffusion | — | 217.21 |
| SDEdit SD1.5 LoRA — ship/syn2real_heavy | Diffusion | — | 219.08 |
| SDEdit SD1.5 LoRA — ship/syn2real | Diffusion | — | 223.21 |
| CE-GAN (patch-level) | GAN | 800 | 339.97 |
| ControlNet | Diffusion | — | 176.61 |

**Reading this table:** among methods with a directly comparable FID, CycleGAN (especially at 200 epochs) produces the closest distributional match to real sonar; SDEdit/SD1.5 LoRA trails by roughly 75–85 FID points; CE-GAN is the weakest, consistent with its very small (89-image) training set.

### 6.2 CycleGAN Variants — LPIPS, Diversity, Cycle Consistency

| Variant | LPIPS (quality) ↓ | Diversity ↑ | Cycle Consistency ↓ |
|---|---|---|---|
| CycleGAN 100 ep | 0.8621 | 0.7909 | 0.3376 |
| CycleGAN 200 ep | 0.8620 | 0.8201 | 0.3874 |

LPIPS is essentially flat between 100 and 200 epochs (no perceptual-quality gain), diversity improves slightly, but cycle consistency actually *worsens* at 200 epochs (likely the reduced λ_cycle, 15→10) — a tradeoff worth flagging rather than reading 200-epoch training as a strict win.

### 6.3 SDEdit — Diversity & Coverage (SD1.5 LoRA, n=200/group)

| Group | Intra-Diversity | Coverage | Density |
|---|---|---|---|
| Real (SCTD+KLSG) baseline | 0.4065 | 1.000 | 6.970 |
| Aug ship [syn2real] | 0.4274 | 0.345 | 0.660 |
| Aug ship [syn2real_heavy] | 0.4240 | 0.330 | 0.630 |
| Aug plane [syn2real] | 0.4082 | 0.300 | 0.765 |
| Aug plane [syn2real_heavy] | 0.4123 | 0.340 | 0.900 |
| **All augmented, combined** | **0.4278** | **0.575** | **2.955** |

Intra-diversity for SDEdit output is close to (even slightly above) the real-data baseline, suggesting no severe mode collapse — but coverage tops out at 57.5%, meaning a sizeable fraction of real images still have no close synthetic neighbor.

### 6.4 Pixel Statistics — How Close Is "Real-Looking"?

| Source | Mean Brightness | Std (Contrast) |
|---|---|---|
| Domain A — Synthetic (S3SIM) | 94.2 | 34.9 |
| Domain B — Real | 69.4 | 49.6 |
| CycleGAN Generated A2B, 100 ep | 80.6 | 42.1 |
| CycleGAN Generated A2B, 200 ep | 65.4 | — |
| SDEdit aug ship [syn2real] | 86.4 | 47.7 |
| SDEdit aug plane [syn2real] | 93.7 | 45.2 |

CycleGAN at 200 epochs gets brightness essentially on target (65.4 vs. real 69.4); SDEdit's outputs stay considerably brighter than real sonar, closer to the synthetic source than to the real target — a sign that the LoRA-adapted SD prior only partially overrides S3SIM's native brightness profile at the strengths tested.

### 6.5 Domain Gap Reduction (PCA Centroid Distance)

| Pipeline | Synthetic→Real | Generated→Real | Reduction |
|---|---|---|---|
| CycleGAN 100 ep | 13.779 | 5.522 | 59.9% |
| CycleGAN 200 ep | 13.725 | 5.442 | 60.3% |

Both CycleGAN checkpoints converge on essentially the same ~60% domain-gap reduction; this metric doesn't distinguish 100 vs. 200 epochs the way FID does. (Equivalent PCA figures for SDEdit/SD2.1/Shadow-CycleGAN/StyleGAN2-ADA were not captured in this round of experiments.)

## 7. Model Size & Compute

| Model | Component | Parameters |
|---|---|---|
| CycleGAN | Generator (×2) | 11.38M each |
| CycleGAN | Discriminator (×2) | 2.76M each |
| Shadow-CycleGAN | All sub-models combined | ~118.39M |
| CE-GAN | Generator | 10.78M |
| CE-GAN | Discriminator | 11.22M |
| StyleGAN2-ADA | Generator (z/w dim = 512) | ~30M+ |
| SD1.5 LoRA | UNet (rank 8) | ~2M trainable |
| SD2.1 LoRA | UNet (rank 64) | ~50M trainable |

LoRA fine-tuning keeps trainable parameters far below the GAN baselines while still leveraging a much larger frozen backbone (SD's ~860M-parameter UNet) — the tradeoff is in adaptation depth (Section 8) rather than raw parameter count.

## 8. Design Rationale

1. **LoRA over full fine-tuning** — preserves SD's general image prior while adapting to the sonar domain on a limited dataset (878 images); rank 8 for SD1.5 (smaller dataset), rank 64 for SD2.1 (larger capacity needed at 768px resolution and a bigger domain gap).
2. **SDEdit (img2img) over text-to-image** — S3SIM already supplies structural priors (object shape, shadow geometry); SDEdit preserves that layout while shifting style/texture toward real sonar.
3. **λ_cycle = 15 for small-dataset CycleGAN runs** — small datasets risk mode collapse; a higher cycle weight keeps input structure intact.
4. **Shadow-CycleGAN's shadow loss** — acoustic shadow is the dominant visual cue separating sonar domains, and standard CycleGAN has no mechanism to supervise it directly; the K-Means shadow loss does.
5. **CE-GAN's WGAN-GP + perceptual loss** — more stable than a vanilla GAN on small, imbalanced datasets; the co-trained classification head keeps the discriminator class-aware.
6. **InceptionV3 for FID** — the standard choice for GAN evaluation, despite a known caveat: it's tuned for natural images, and a sonar-fine-tuned ResNet18 would likely be more appropriate, but no such pretrained model was available for this study.
7. **LPIPS over SSIM** — SSIM needs pixel-level alignment, which an unpaired Sim2Real setup doesn't have; LPIPS measures perceptual realism in feature space without requiring pairing.

## 9. Key Findings

- **Best overall FID:** CycleGAN at 200 epochs (140.56), the strongest distributional match to real sonar among all methods with a captured FID.
- **Best diffusion-family FID:** SDEdit SD1.5 LoRA, plane/syn2real_heavy (215.28) — still roughly 75 FID points behind the best GAN result.
- **Domain gap reduction is consistent across CycleGAN checkpoints** at ~60% (PCA centroid distance), regardless of 100 vs. 200 epochs.
- **Diversity is comparable to real data for SDEdit** (intra-diversity ≈0.43 vs. real baseline ≈0.41), but **coverage is incomplete** (57.5%) — augmented data clusters around only part of the real distribution.
- **More training epochs ≠ uniformly better.** CycleGAN's FID improves from 100→200 epochs, but cycle consistency LPIPS gets *worse* over the same span, likely tied to the reduced cycle-loss weight.
- **Brightness/contrast match is method-dependent.** CycleGAN 200ep nearly matches real brightness (65.4 vs. 69.4); SDEdit outputs remain notably brighter, closer to the synthetic source.

## 10. Limitations & Open Issues

- Shadow-CycleGAN's shadow-specific LPIPS + IoU evaluation cell did not capture a final numeric output in this run; results live only in `sh_OUTPUTS_10/` pending recovery.
- CE-GAN's FID (339.97) is poor, attributable to the very small training set (89 patches at 64px) — perceptual loss prevented full collapse but didn't yield high-fidelity samples. Downstream classifier accuracy with CE-GAN augmentation (0.74) was marginally *below* the no-augmentation baseline (0.75), despite a comparable Class-0 F1.
- SD2.1 LoRA's latest checkpoint (epoch 40) has unconfirmed convergence; later epochs were not trained/evaluated.
- StyleGAN2-ADA and ControlNet (both pre- and post-fine-tuning) were not included in the quantitative FID/LPIPS comparison in this round — only qualitative/structural assessment is available for ControlNet, and StyleGAN2-ADA's generated set was assembled but not benchmarked against the same metric suite.
- FID's use of an ImageNet-pretrained InceptionV3 backbone is a known mismatch for acoustic sonar imagery; absolute FID values across families should be read as directionally informative rather than as an absolute realism score.
- This repository contains code only — no datasets and no pre-computed metric files are checked in. All numbers in [Section 6](#6-comparative-results) come from the authors' own local runs and are reported for documentation purposes; reproducing them exactly requires sourcing the same dataset versions (see [§3.3](#33-obtaining-the-data)) and re-running `QuantitativeMetricsFinal.ipynb`.

### Getting started

Each method is implemented as a standalone notebook (see [Repository Structure](#2-repository-structure)); there is no single shared entry-point script, and **no data ships with this repo**.

1. Install the libraries listed above into your environment.
2. Download SCTD and KLSG from the links in [§3.3](#33-obtaining-the-data), and source S3SIM separately; recreate the `SIM2REAL/fake/`, `SIM2REAL/real/`, `SCTD/`, and `KLSG/` folders locally as shown in [Repository Structure](#2-repository-structure).
3. Open the relevant notebook under `Diffusion/` or `GAN/` in Jupyter or Colab (notebooks marked "Colab" in their filename were run there due to VRAM constraints), and point its data paths at the folders you just created.
4. For metrics, run `Diffusion/ControlNet/QuantitativeMetricsFinal.ipynb`, which contains the full FID/LPIPS/diversity/coverage evaluation suite. This regenerates `metric_outputs/` locally — the numbers reported in [Section 6](#6-comparative-results) reflect the authors' own run and are not checked into the repo, so re-running may shift them slightly depending on dataset version and sampling.
